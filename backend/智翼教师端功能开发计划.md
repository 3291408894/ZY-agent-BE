## 总体概述

### 当前状态
智翼目前是一个**纯学生端**的 K12 AI 学习助手，技术栈：

| 层级 | 技术 | 状态 |
|------|------|:----:|
| 前端 | Vue 3.4 + TypeScript + Vite 5 + Element Plus + ECharts + Pinia | ✅ 完成 |
| 后端 | Python 3.12 + FastAPI + SQLAlchemy 2.0 + MySQL + Redis | ✅ 完成 |
| AI | LLM Client (DeepSeek) + LangChain Agent + SSE 流式 | ✅ 完成 |
| 角色 | **无角色系统，纯学生视角** | ❌ 缺失 |

### 开发目标
新增**三个教师端核心功能模块**，从底层角色系统到上层业务功能，完整覆盖教师日常教学场景。

### 技术约束
- **严格遵守现有代码模式**（SQLAlchemy 2.0 Mapped、Pydantic v2、Pinia Composition API、Vue `<script setup lang="ts">`）
- **复用现有基础设施**（JWT 认证、SSE 流式、文件上传解析、LLM Client）
- **保持前后端一致的数据契约**

## 一、角色系统改造

> ⚠️ **所有教师功能的前置依赖，必须最先实施**

### 1.1 数据库变更

```sql

  ADD COLUMN `role` VARCHAR(20) NOT NULL DEFAULT 'student' 
  COMMENT '用户角色: student-学生, teacher-教师, admin-管理员'
```

### 1.2 后端改造

| 文件 | 操作 | 关键内容 |
|------|:--:|------|

| `backend/app/schemas/user.py` | ✏️修改 | 新增 `class UserRole(str, Enum): STUDENT="student", TEACHER="teacher", ADMIN="admin"`，`RegisterRequest`/`UserResponse` 加 `role` 字段 |

```python
# deps.py 新增依赖
async def get_current_teacher(
    current_user: User = Depends(get_current_user)
) -> User:
    if current_user.role not in ("teacher", "admin"):
        raise HTTPException(
            status_code=403,
            detail={"code": ErrorCode.FORBIDDEN, "message": "仅教师可访问此功能", "detail": None}
        )
    return current_user
```

## 二、智能教案生成

> 教师输入学科/年级/章节/课时数 → AI 流式生成结构化教案 → 历史管理 → 导出 Word/PDF

### 2.1 数据库表 — `lesson_plans`

```sql
CREATE TABLE `lesson_plans` (
  `id`             CHAR(36)     NOT NULL COMMENT '教案ID',
  `user_id`        CHAR(36)     NOT NULL COMMENT '所属教师ID',
```

## 三、试卷生成器

> 教师配置题型分布/难度配比 → AI 流式生成完整试卷 → A4纸预览 → 答题卡自动生成 → 一键导出可打印版

### 3.1 数据库表 — `exam_papers`

```sql
CREATE TABLE `exam_papers` (
  `id`                 CHAR(36)     NOT NULL COMMENT '试卷ID',
  `user_id`            CHAR(36)     NOT NULL COMMENT '创建教师ID',
  `title`              VARCHAR(200) NOT NULL COMMENT '试卷标题',
  `subject`            VARCHAR(50)  NOT NULL COMMENT '学科',
  `grade`              VARCHAR(50)  NOT NULL COMMENT '年级',
  `exam_type`          VARCHAR(30)  NOT NULL COMMENT '考试类型: unit_test/midterm/final',
  `total_score`        INT          NOT NULL COMMENT '总分',
  `difficulty_ratio`   JSON         NOT NULL COMMENT '难度配比: {"easy":30,"medium":50,"hard":20}',
  `question_structure` JSON         NOT NULL COMMENT '题型分布: [{"type":"选择","count":10,"score_per":3,"subtotal":30}]',
  `content`            JSON         NOT NULL COMMENT '试卷正文JSON: {header,sections[{title,instructions,questions[]}],answer_key,scoring_guide}',
  `answer_sheet`       JSON         NULL     COMMENT '答题卡JSON（服务端自动生成）',
  `export_url`         VARCHAR(500) NULL     COMMENT '导出URL',
  `export_format`      VARCHAR(20)  NULL     COMMENT '导出格式',
  `status`             VARCHAR(20)  NOT NULL DEFAULT 'completed',
  `error_message`      TEXT         NULL,
  `created_at`         DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
  `updated_at`         DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  INDEX `ix_ep_user_id` (`user_id`),
  INDEX `ix_ep_subject` (`subject`),
  INDEX `ix_ep_exam_type` (`exam_type`),
  INDEX `ix_ep_created_at` (`created_at`),
  CONSTRAINT `fk_ep_user` FOREIGN KEY (`user_id`) REFERENCES `users`(`id`) ON DELETE CASCADE
) ENGINE=InnoDB CHARSET=utf8mb4 COMMENT='AI试卷生成记录';
```

### 3.5 关键技术方案

**A4 纸预览**：
```scss
.exam-preview__paper {
  width: 210mm;
  min-height: 297mm;
  padding: 20mm 15mm 25mm;
  background: #fff;
  box-shadow: 0 2px 12px rgba(0,0,0,0.12);
  font-family: 'SimSun', serif;
  font-size: 14px;
  line-height: 1.8;

  @media print {
    box-shadow: none; padding: 0; width: auto;
    page-break-after: always;
  }
}
```

**答题卡自动生成**：Service 层根据 `content.sections` 直接构建，不依赖 AI（避免格式不稳定）。

**Word 导出**：python-docx 设置 A4 页面 + 密封线 + 分值标注 + 每道题留白答题区 + 分页断点。

**PDF 导出**：可选 LibreOffice headless 转换或 WeasyPrint 渲染 HTML 模板。


## 六、关键技术决策

| 决策点 | 选择 | 理由 |
|------|------|------|
| 教案/试卷存储 | MySQL JSON 列 | 结构固定但嵌套深，避免多表 JOIN，MySQL 8.0+ JSON 支持成熟 |
| SSE 状态机命名 | `config → generating → result` | 与现有 `ExerciseView` 模式完全一致 |
| AI Prompt 格式 | System Prompt（角色）+ User Template（.format()） | 与现有 `summary.py` / `exercise_gen.py` 模式一致 |
| 导出方案 | python-docx（主要）+ LibreOffice（转PDF） | 纯 Python 无系统依赖，教师可编辑 Word，PDF 通过转换 |
| 答题卡生成 | 服务端自动构建，不依赖 AI | AI 输出格式不稳定，服务端保证与试卷一致性 |
| 文件存储 | 本地文件系统 + 按 user_id 分层 | 初期量级够用，后续平滑迁移 MinIO/S3 |
| 搜索 | MySQL InnoDB FULLTEXT 索引 | 初期不引入 ES，标题+描述+关键词足够 |
| A4 预览 | CSS `@media print` + `@page { size: A4 }` | 纯 CSS，零依赖，打印/预览一体化 |
| 角色鉴权 | FastAPI Depends 依赖注入链 | 与现有 `get_current_user()` 模式一致，无侵入 |

## 八、验证方案

### 8.1 角色系统验证
1. 教师注册 → 登录 → 侧边栏显示「智能教案」「试卷生成」「资源库」
2. 学生注册 → 登录 → 侧边栏显示「学习仪表盘」「AI助手」「课文总结」等学生菜单
3. 学生直接访问 `/teacher/lesson-plan` → 重定向到 `/dashboard`

### 8.2 教案生成验证
1. 输入：学科=数学、年级=八年级、章节=勾股定理、课时=2 → 点击生成
2. SSE 流式展示过程中：看到「正在分析教学目标...」「正在设计教学过程...」等阶段提示
3. 生成完成：教学目标/重难点/教学过程表格/板书设计/反思模板 全部正确渲染
4. 导出 Word → 打开确认排版正确（标题居中、表格有边框线、各部分有层级标题）
5. 历史列表 → 可查看/删除

### 8.3 试卷生成验证
1. 配置：高一数学期中考试，总分 150，简单30%+中等50%+困难20%
2. 题型：选择10道×5分 + 填空5道×6分 + 简答4道×10分 + 综合2道×15分
3. 生成 → A4预览效果正确 → 打印预览分页无误
4. 答题卡：学生信息区、选择题填涂区、填空题作答区、简答/综合留白区均正确
5. 导出 Word → 可打印 → 密封线位置正确

### 8.4 资源库验证
1. 上传一个课件 PPT → 填写标题/学科/年级/类型/标签 → 上传成功
2. 资源广场可见 → 搜索文件名/学科筛选均正确
3. 点击下载 → 文件正确下载 → 下载计数+1
4. 收藏 → 列表中显示已收藏状态 → 取消收藏成功
5. 上传者删除 → 广场中该资源消失
